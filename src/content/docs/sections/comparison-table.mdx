The **ComparisonTable** allows you to compare scope, capabilities, or delivery options across reusable project variants.

## Usage

```astro
import ComparisonTable from '~/components/sections/ComparisonTable.astro';

const plans = [
    { name: 'Starter', description: 'Single-page presence', key: 'starter' },
    { name: 'Company', description: 'Common website scope', popular: true, key: 'company' },
    { name: 'Custom', description: 'Tailored web solution', key: 'custom' }
];

const features = [
    { category: 'Project Scope', items: [
        { name: 'Pages or sections', starter: 'Focused', company: 'Several', custom: 'As needed' },
        { name: 'Editable content', starter: false, company: true, custom: true }
    ]},
    { category: 'Care', items: [
        { name: 'Launch support', starter: true, company: true, custom: true },
        { name: 'Future improvements', starter: 'Optional', company: 'Planned', custom: 'Roadmap' }
    ]}
];

<ComparisonTable 
  plans={plans}
  features={features}
/>
```

## Props

### ComparisonTable

| Prop | Type | Description |
| :--- | :--- | :--- |
| `plans` | `Plan[]` | Array of comparison option objects (max 3 recommended). |
| `features` | `FeatureCategory[]` | Categorized list of features to compare. |

### Data Structures

```ts
interface Plan {
  name: string;
  description: string;
  popular?: boolean;
}

interface FeatureCategory {
  category: string;
  items: FeatureItem[];
}

interface FeatureItem {
  name: string;
  // Key must match the option key
  // Value can be string, number, or boolean
  [planName: string]: string | number | boolean;
}
```
