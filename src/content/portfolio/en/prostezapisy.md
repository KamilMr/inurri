## Project goal

ProsteZapisy.pl was created as a simple booking system for people who work with clients by appointment. The goal was to build a tool that lets users accept online bookings without their own website, without client accounts, and without complex setup.

The user shares a link to their calendar. The client chooses an available time, leaves the required details, and the person accepting bookings receives an email notification.

## Scope

- product website implementation,
- frontend application built with Next.js,
- backend for bookings and user accounts,
- public calendar link with available time slots,
- booking form without client login,
- email notifications after a new booking,
- simple panel for configuring availability and account details.

## Approach

The project was built around one main problem: accepting bookings without a large system and without needing a separate website. The interface focuses on the essential flow: setting available times, sharing a link, and receiving information about a new reservation.

Next.js was used for the frontend and application layer. The backend handles users, availability, bookings, and notification delivery.

## Result

The result is a complete web application with a product website, user panel, and public booking view for clients. The system makes it possible to launch a simple online calendar and start accepting bookings without extra infrastructure on the user's side.
